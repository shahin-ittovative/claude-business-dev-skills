# Concept Sheet

A concise document that outlines the core idea of a product, project, or business initiative. It is used to present a proposal at an early stage to stakeholders, clients, investors, or internal teams before developing a full business plan or prototype.

| Section | What to capture | Answer |
|---|---|---|
| Title / Name | A short, clear name for the concept (for example, product name). |  |
| Overview / Summary | A 2 to 4 line high-level description of the concept: what it is, what problem it solves, and for whom. |  |
| Problem Statement | Clearly state the gap or pain point in the market that this idea addresses. Include data if possible. |  |
| Proposed Solution / Idea Description | A detailed explanation of the idea, product, or service: what it does, how it works, and why it matters. |  |
| Target Audience / Users | Define the primary users or customer segments. Consider demographics (B2B vs B2C, company size, role), firmographics (industry, revenue, location), behavior (early adopters, existing customers). Use personas and the JTBD (Jobs to be Done) framework. |  |
| Unique Value Proposition (UVP) | What makes this concept better or different from alternatives: the problem solved, why it is better or different, and the core benefits to the customer. |  |
| Key Features | List of core features or components that deliver the solution. |  |
| Business Model | Brief explanation of how revenue is generated (subscription, pay-per-use, B2B licensing, and so on). |  |
| Expected Benefits / Impact | Quantify or describe the impact (cost savings, faster onboarding, reduced churn). |  |
| Success Metrics | How success will be measured. |  |
| Technology / Tools Used | Critical technologies, platforms, or APIs involved (frontend, backend, hosting, third-party services). |  |
| Competitor Landscape | Reference the Market Comparison framework (file 06). |  |
