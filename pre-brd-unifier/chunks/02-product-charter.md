# Product Charter

A foundational document that defines the purpose, goals, scope, stakeholders, and boundaries of a product or project. It aligns the team, sponsors, and stakeholders before development begins. It is often used in product management, project kickoffs, or agile initiatives to make sure everyone is on the same page.

| Section | What to capture | Answer |
|---|---|---|
| Product Name | The official name of the product, reflecting its purpose. |  |
| Product Manager | The person responsible for leading the product's development, roadmap, and execution. |  |
| Date | Date of the charter (YYYY-MM-DD). |  |
| Stakeholders | Key internal and external parties involved (sponsor, product manager, engineering lead, UX designer, pilot clients). |  |
| Problem Statement | What problem or pain point are we solving? See Concept Sheet, Problem Statement. |  |
| Purpose / Vision | Why this product exists and the long-term goal it supports. |  |
| Objectives / Goals | What we aim to achieve, stated as SMART goals. |  |
| Scope (MVP) | What is included in this phase of the product. |  |
| Out of Scope (MVP) | What is excluded in this phase of the product. |  |
| Key Features / Deliverables | Core features or deliverables for this phase. See Concept Sheet, Key Features. |  |
| Limitations | Known limitations. |  |
| Risks | Known potential risks. |  |
| Success Metrics | How success will be measured. See Concept Sheet, Success Metrics. |  |
| Timeline / Milestones | High-level phases or key dates. |  |
